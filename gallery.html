❓ How adp-show-graphics works

Events

Each event has its own isolated image library and layouts. Switch events from the top dropdown. Add events with + Event.

Two Layers

Graphics — main show graphics. Bug/QR — persistent lower-third overlays. Each layer has H + V format slots = 4 output URLs total.

Send to Slot

Select target slot (Graphics H/V, Bug H/V) in the bar above the grid. Click any image to send it to that slot.

SEND LIVE

Selecting an image only updates the gallery preview. Click SEND LIVE to push it to the output URL. Toggle again to go transparent.

Output URLs

Add each URL as a browser source in vMix. Append ?event=ID to target a specific event. URLs shown in each slot header.

Resolution

Set H and V canvas size independently in ⚙ Settings per event. Presets: 4K, 1440p, 1080p. Changes affect output pages immediately.

Image Library

All uploaded originals. Search by name. Filter by tag. Drag images to reorder. Tags are persistent — name images well before uploading.

Upload

Click 📤 Upload or drag files from Finder anywhere onto the window. Assign categories and auto-tag from filename during upload.

Tags

Assign categories to images via the 🏷 tag icon on each card. Manage categories and auto-tag keyword rules in Tag Management.

Layouts

Save current slot state as a layout. Recall later with ↩. Layout stores graphics + bugs slots for both H and V.

Background References

Upload slide/screen reference images for the slot preview background. Tagged background. Visible only in gallery — never in output.

Quick trigger (vMix)

GET /go?apikey=X&event=E&layer=graphics&slot=h&key=K — set and go live in one call, usable from vMix scripting.
